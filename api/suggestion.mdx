MoodJourney provides AI-powered features through the Gemini API, including personalized suggestions for journal entries and an interactive chat assistant.

## generate_suggestion_cmd

Generates three actionable suggestions based on a journal entry's title and content. The suggestions are tailored to the emotional tone of the entry.

### Function signature

```rust
async fn generate_suggestion_cmd(
    entry_title: Option<String>,
    entry_content: Option<String>,
) -> Result<String, String>
```

### Parameters

<ParamField path="entryTitle" type="string | null">
  The title of the journal entry. Can be `null` if only content is provided.
</ParamField>

<ParamField path="entryContent" type="string | null">
  The main content of the journal entry. Can be `null` if only title is provided.
  
  **Note**: Content is automatically truncated to 700 characters before being sent to the API.
</ParamField>

### Returns

<ResponseField name="suggestions" type="string">
  A formatted string containing three suggestions, each on its own line starting with a dash (e.g., `- Suggestion 1\n- Suggestion 2\n- Suggestion 3`)
</ResponseField>

<ResponseField name="error" type="string">
  Error message if generation fails. Common errors:
  - Both title and content are empty
  - API key not configured
  - Network request failed
  - API returned empty response
</ResponseField>

### Usage example

```javascript
import { invoke } from "@tauri-apps/api/core";

const journalContent = "Today was a bit overwhelming. I had a lot of meetings and didn't get as much done on my main project as I hoped. Feeling a little drained now.";

try {
  const suggestions = await invoke("generate_suggestion_cmd", {
    entryTitle: "Journal Entry",
    entryContent: journalContent
  });
  
  console.log("AI Suggestions:", suggestions);
  // Output:
  // - Perhaps take 15 minutes for a quick walk or some stretching to clear your head and recharge.
  // - Before tomorrow, could you identify one or two key tasks for your main project to focus on first?
  // - Remember to acknowledge what you *did* manage today, even with many meetings.
} catch (error) {
  console.error("Failed to generate suggestions:", error);
}
```

### Implementation details

- **API**: Google Gemini 2.0 Flash (`gemini-2.0-flash:generateContent`)
- **Model parameters**:
  - Temperature: `0.7`
  - Top P: `0.9`
  - Top K: `40`
  - Max tokens: `150`
- **Content limit**: First 700 characters of content (see `main.rs:282-290`)
- **Prompt engineering**: Includes examples for different emotions (joy, sadness, anger) to guide the model

### Prompt structure

The command builds a detailed prompt with examples (see `main.rs:240-297`):

## chat_assistant_cmd

<Note>
This command is referenced in `main.rs` but the actual command name used in the code is `chat_with_moodjourney_cmd` (see `main.rs:339-433`). The documentation here uses the canonical name from the source.
</Note>

Engages in a conversational chat with the MoodJourney AI assistant. The assistant has access to all past journal entries and maintains conversation history across multiple turns.

### Function signature

```rust
async fn chat_with_moodjourney_cmd(
    app_db_path: State<'_, PathBuf>,
    user_message: String,
    session_id_option: Option<String>,
) -> Result<ChatCompletionResponse, String>
```

### Parameters

<ParamField path="userMessage" type="string" required>
  The user's message to send to the assistant
</ParamField>

<ParamField path="sessionIdOption" type="string | null">
  The chat session ID. Pass `null` to start a new conversation, or provide an existing session ID to continue a conversation.
</ParamField>

### Returns

<ResponseField name="response" type="ChatCompletionResponse">
  <ResponseField name="assistant_response" type="string">
    The AI assistant's response to the user's message
  </ResponseField>
  
  <ResponseField name="session_id" type="string">
    The session ID for this conversation (use this for subsequent messages)
  </ResponseField>
</ResponseField>

### Usage example

```javascript
import { invoke } from "@tauri-apps/api/core";

let currentSessionId = null;

// Start a new conversation
try {
  const response = await invoke("chat_with_moodjourney_cmd", {
    userMessage: "How have I been feeling this week?",
    sessionIdOption: currentSessionId
  });
  
  console.log("Assistant:", response.assistant_response);
  currentSessionId = response.session_id; // Save for next message
} catch (error) {
  console.error("Chat failed:", error);
}

// Continue the conversation
const followUp = await invoke("chat_with_moodjourney_cmd", {
  userMessage: "What can I do to feel better?",
  sessionIdOption: currentSessionId // Continue same session
});

console.log("Assistant:", followUp.assistant_response);
```

### Implementation details

- **API**: Google Gemini 2.0 Flash
- **Model parameters**:
  - Temperature: `0.75`
  - Top P: `0.9`
  - Top K: `40`
  - Max tokens: `600`
- **Context**: The first message in a session includes all past journal entries as context
- **Session management**: Sessions are stored in the database with auto-generated titles
- **Message history**: Full conversation history is sent with each request for context

### Context injection

On the first message in a session, the assistant receives (see `main.rs:367-399`):

```javascript
// System prompt
You are a helpful journaling assistant called "MoodJourney". 
The current date is {date}.
You have access to the user's past journal entries for context...

// All past journal entries
Date: 2025-04-20
Content: {entry content without emotion/suggestion metadata}
---
Date: 2025-04-19
Content: ...

// User's message
User: How have I been feeling this week?
```

### Session title generation

The session title is automatically set to the first 50 characters of the user's first message:

```rust

    let new_title: String = content.chars().take(50).collect();
    let final_title = if content.chars().count() > 50 {

        new_title 
    };
}
```

## load_chat_sessions

Retrieves all chat sessions from the database, ordered by creation date (newest first).

### Function signature

```rust
async fn load_chat_sessions(
    app_db_path: State<'_, PathBuf>
) -> Result<Vec<ChatSession>, String>
```

### Returns

<ResponseField name="sessions" type="ChatSession[]">
  Array of chat sessions, each containing:
  - `id` (string) - Unique session identifier
  - `created_at` (string) - ISO 8601 timestamp
  - `last_modified_at` (string) - ISO 8601 timestamp of last update
  - `title` (string | null) - Session title (first 50 chars of first message)
</ResponseField>

### Usage example

```javascript
import { invoke } from "@tauri-apps/api/core";

try {
  const sessions = await invoke("load_chat_sessions");
  console.log("Chat sessions:", sessions);
  // [{ id: "abc123", created_at: "2025-04-20T10:30:00Z", last_modified_at: "2025-04-20T10:35:00Z", title: "How have I been feeling..." }, ...]
} catch (error) {
  console.error("Failed to load sessions:", error);
}
```

## load_messages_for_session_cmd

Retrieves all messages for a specific chat session, ordered chronologically.

### Function signature

```rust
async fn load_messages_for_session_cmd(
    app_db_path: State<'_, PathBuf>,
    session_id: String
) -> Result<Vec<ChatMessage>, String>
```

### Parameters

<ParamField path="sessionId" type="string" required>
  The unique session ID to retrieve messages for
</ParamField>

### Returns

<ResponseField name="messages" type="ChatMessage[]">
  Array of messages in chronological order, each containing:
  - `id` (number) - Unique message identifier
  - `session_id` (string) - Parent session ID
  - `sender` (string) - Either "user" or "assistant"
  - `content` (string) - Message text
  - `timestamp` (string) - ISO 8601 timestamp
</ResponseField>

### Usage example

```javascript
import { invoke } from "@tauri-apps/api/core";

try {
  const messages = await invoke("load_messages_for_session_cmd", {
    sessionId: "abc123"
  });
  
  messages.forEach(msg => {
    console.log(`${msg.sender}: ${msg.content}`);
  });
} catch (error) {
  console.error("Failed to load messages:", error);
}
```

## delete_chat_session_cmd

Deletes a chat session and all its associated messages from the database.

### Function signature

```rust
async fn delete_chat_session_cmd(
    app_db_path: State<'_, PathBuf>,
    session_id: String
) -> Result<(), String>
```

### Parameters

<ParamField path="sessionId" type="string" required>
  The unique session ID to delete
</ParamField>

### Returns

Returns `Ok(())` on success or an error string on failure.

### Usage example

```javascript
import { invoke } from "@tauri-apps/api/core";

try {
  await invoke("delete_chat_session_cmd", {
    sessionId: "abc123"
  });
  console.log("Session deleted successfully");
} catch (error) {
  console.error("Failed to delete session:", error);
}
```

<Warning>
This operation is permanent and cannot be undone. All messages in the session will be deleted via CASCADE DELETE (see `database.mdx` for schema details).
</Warning>
