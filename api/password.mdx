MoodJourney provides optional PIN-based security to protect journal entries. The PIN system uses SHA-256 hashing and persistent storage.

## is_pin_set_cmd

Checks whether a PIN has been configured for the application.

### Function signature

```rust
fn is_pin_set_cmd(
    app_password_state: State<'_, Mutex<PasswordState>>
) -> bool
```

### Parameters

No parameters required.

### Returns

<ResponseField name="isSet" type="boolean">
  `true` if a PIN is currently set, `false` otherwise
</ResponseField>

### Usage example

```javascript
import { invoke } from "@tauri-apps/api/core";

const pinIsSet = await invoke('is_pin_set_cmd');

if (pinIsSet) {
  console.log("PIN protection is enabled");
} else {
  console.log("No PIN configured");
}
```

### Typical workflow

From `App.jsx:139-160`, the app checks PIN status on startup:

```javascript
const checkPinStatus = async () => {
  const pinIsCurrentlySet = await invoke('is_pin_set_cmd');
  setIsPinSet(pinIsCurrentlySet);
  
  if (pinIsCurrentlySet) {
    const appIsLocked = await invoke('is_locked_cmd');
    if (appIsLocked) {
      setIsAppLocked(true);
      setPinAction('unlock');
      setShowPinModal(true);
    }
  }
};
```

## check_password_attempt_cmd

Verifies a PIN attempt against the stored hash. If successful, automatically unlocks the app.

### Function signature

```rust
fn check_password_attempt_cmd(
    app_password_state: State<'_, Mutex<PasswordState>>,
    password_str: String,
) -> bool
```

### Parameters

<ParamField path="passwordStr" type="string" required>
  The PIN to verify (typically a 4-digit string)
</ParamField>

### Returns

<ResponseField name="isValid" type="boolean">
  `true` if the PIN is correct, `false` otherwise
</ResponseField>

### Side effects

<Note>
If the PIN is correct, the app is **automatically unlocked** (the `locked` state is set to `false`).
</Note>

### Usage example

```javascript
import { invoke } from "@tauri-apps/api/core";

const userPin = "1234";

try {
  const isValid = await invoke('check_password_attempt_cmd', { 
    passwordStr: userPin 
  });
  
  if (isValid) {
    console.log("PIN verified. App unlocked.");
    setIsAppLocked(false);
  } else {
    console.log("Invalid PIN. Try again.");
    setPinError("Invalid PIN. Try again.");
  }
} catch (error) {
  console.error("PIN verification failed:", error);
}
```

### Implementation details

- Hashes the attempted PIN using SHA-256 (see `password.rs:77-81`)
- Compares the hash with the stored password hash
- If match, sets `locked = false` and persists to disk
- Implementation in `password.rs:88-98`

## set_new_password_cmd

Sets or changes the application PIN. The new PIN is hashed before storage.

### Function signature

```rust
fn set_new_password_cmd(
    app_password_state: State<'_, Mutex<PasswordState>>,
    password_str: String,
) -> ()
```

### Parameters

<ParamField path="passwordStr" type="string" required>
  The new PIN to set. Pass an empty string to remove the PIN entirely.
</ParamField>

### Returns

No return value (void).

### Side effects

<Warning>
Setting a new PIN automatically **locks** the app (`locked = true`). The user must use `check_password_attempt_cmd` to unlock.
</Warning>

### Usage example

```javascript
import { invoke } from "@tauri-apps/api/core";

// Set a new PIN
const newPin = "1234";

try {
  await invoke('set_new_password_cmd', { passwordStr: newPin });
  console.log("PIN created successfully");
  
  // App is now locked, set lock state to true
  await invoke('set_locked_explicit_cmd', { locked: true });
  setIsAppLocked(true);
} catch (error) {
  console.error("Failed to set PIN:", error);
}

// Remove PIN by passing empty string
await invoke('set_new_password_cmd', { passwordStr: "" });
console.log("PIN removed");
```

### Implementation details

- If `password_str` is empty, removes the PIN and unlocks the app
- Otherwise, hashes the PIN with SHA-256 and stores the hash
- Automatically sets `locked = true` when a non-empty PIN is set
- Persists to `{app_config_dir}/password.json`
- See `password.rs:100-110`

## is_locked_cmd

Checks if the application is currently locked.

### Function signature

```rust
fn is_locked_cmd(
    app_password_state: State<'_, Mutex<PasswordState>>
) -> bool
```

### Parameters

No parameters required.

### Returns

<ResponseField name="isLocked" type="boolean">
  `true` if the app is locked (and a PIN is set), `false` otherwise
</ResponseField>

### Important behavior

<Note>
If **no PIN is set**, this command always returns `false`, even if the internal `locked` flag is `true`. The app can only be locked if a PIN exists.
</Note>

From `password.rs:116-123`:

```rust
pub fn is_locked_internal(&self) -> bool {
    if self.is_pin_set_internal() {
        self.password_data.locked
    } else {
        false  // Can't be locked without a PIN
    }
}
```

### Usage example

```javascript
import { invoke } from "@tauri-apps/api/core";

const isLocked = await invoke('is_locked_cmd');

if (isLocked) {
  console.log("App is locked. PIN required.");
  // Show PIN unlock modal
} else {
  console.log("App is unlocked");
}
```

## set_locked_explicit_cmd

Manually locks or unlocks the application (only effective if a PIN is set).

### Function signature

```rust
fn set_locked_explicit_cmd(
    app_password_state: State<'_, Mutex<PasswordState>>,
    locked: bool,
) -> ()
```

### Parameters

<ParamField path="locked" type="boolean" required>
  `true` to lock the app, `false` to unlock it
</ParamField>

### Returns

No return value (void).

### Usage example

```javascript
import { invoke } from "@tauri-apps/api/core";

// Lock the app
await invoke('set_locked_explicit_cmd', { locked: true });
console.log("App locked");

// Unlock the app (bypasses PIN check - use with caution!)
await invoke('set_locked_explicit_cmd', { locked: false });
console.log("App unlocked");
```

### Security note

<Warning>
This command **bypasses PIN verification**. In typical usage, unlocking should only happen through `check_password_attempt_cmd`, which validates the PIN first.

Use `set_locked_explicit_cmd` for:
- Locking the app manually (e.g., on idle timeout)
- Testing purposes
- **Not** for production unlock flows
</Warning>

## delete_pin_cmd

Completely removes the PIN and unlocks the application.

### Function signature

```rust
fn delete_pin_cmd(
    app_password_state: State<'_, Mutex<PasswordState>>
) -> ()
```

### Parameters

No parameters required.

### Returns

No return value (void).

### Side effects

- Removes the stored password hash
- Sets `locked = false`
- Persists changes to disk

### Usage example

```javascript
import { invoke } from "@tauri-apps/api/core";

try {
  await invoke('delete_pin_cmd');
  console.log("PIN deleted successfully");
  
  setIsPinSet(false);
  setIsAppLocked(false);
} catch (error) {
  console.error("Failed to delete PIN:", error);
}
```

### When to use

This is typically called from the Settings page when the user wants to disable PIN protection entirely.

### Implementation

From `password.rs:125-129`:

```rust
pub fn delete_pin_internal(&mut self) {
    self.password_data.password_hash = None;
    self.password_data.locked = false;
    self.save();
}
```

## Storage and persistence

All PIN data is stored in a JSON file at `{app_config_dir}/password.json`:

```json
{
  "password_hash": "5e884898da28047151d0e56f8dc6292773603d0d6aabbdd62a11ef721d1542d8",
  "locked": true
}
```

### Security details

- **Hashing algorithm**: SHA-256
- **Storage location**: App config directory (platform-specific)
- **Plaintext storage**: No, only the hash is stored
- **Salt**: Not currently implemented (future improvement)

<Warning>
PINs are hashed with SHA-256 **without a salt**. While this prevents plaintext storage, it makes the system vulnerable to rainbow table attacks. For production use, consider adding a salt or using a key derivation function like bcrypt or Argon2.
</Warning>

### File location by platform

- **Linux**: `~/.config/{app.name}/password.json`
- **macOS**: `~/Library/Application Support/{app.name}/password.json`
- **Windows**: `C:\Users\{user}\AppData\Roaming\{app.name}\password.json`

## Complete workflow example

Here's a complete PIN management flow:

```javascript
import { invoke } from "@tauri-apps/api/core";

// 1. Check if PIN is set on app startup
const pinIsSet = await invoke('is_pin_set_cmd');

if (pinIsSet) {
  // 2. Check if app is locked
  const isLocked = await invoke('is_locked_cmd');
  
  if (isLocked) {
    // 3. Show PIN entry modal
    const userPin = await promptForPin(); // Your UI logic
    
    // 4. Verify PIN
    const isValid = await invoke('check_password_attempt_cmd', { 
      passwordStr: userPin 
    });
    
    if (isValid) {
      console.log("Unlocked successfully");
    } else {
      console.log("Invalid PIN");
    }
  }
} else {
  console.log("No PIN configured. App is accessible.");
}

// To create a PIN:
const newPin = "1234";
await invoke('set_new_password_cmd', { passwordStr: newPin });
await invoke('set_locked_explicit_cmd', { locked: true });

// To delete the PIN:
await invoke('delete_pin_cmd');
```
